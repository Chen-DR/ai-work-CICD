# 脚本管理功能开发需求

## 功能概述

为系统新增「脚本管理」模块，允许用户通过 Web 界面管理和执行 Shell 脚本，
并实时查看脚本运行日志输出。

---

## 一、脚本管理（CRUD）

### 上传脚本
- 支持通过「文件选择」或「拖拽上传」方式上传 `.sh` / `.py` / `.bash` 文件
- 上传时自动读取文件名作为脚本名称，支持用户修改
- 支持填写脚本描述（可选）
- 上传成功后脚本显示在列表中

### 删除脚本
- 列表每行提供「删除」按钮
- 删除前弹出确认对话框，防止误操作
- 删除后从列表中移除

### 查看脚本内容
- 点击脚本名称或「查看」按钮，在右侧或弹窗中展示脚本内容
- 使用代码编辑器组件展示（推荐 CodeMirror 或 Monaco Editor）
- 支持语法高亮（Shell、Python）

### 在线编辑脚本
- 代码编辑器支持直接修改内容
- 提供「保存」按钮，将修改同步到服务器
- 保存前提示用户确认（若内容有改动）
- 支持快捷键 Ctrl+S 触发保存

---

## 二、脚本执行

### 执行配置
页面右侧提供固定的「执行配置」面板，用户在执行前配置以下参数：

- **目标服务器**：下拉选择要在哪台服务器上执行脚本；下拉项显示服务器名称、IP、
  在线状态（绿点/红点）及当前负载；服务器列表从系统已注册的服务器中读取
- **工作目录（cwd）**：手动输入或点击目录浏览按钮选择脚本执行时的工作目录；
  下方展示最近使用过的路径（最多 3 条），点击可快速填入
  - 工作目录候选从「服务器详情 -> 允许的工作目录」读取
  - 用途为 `script` 或 `general` 的目录会作为脚本执行快捷目录展示
- **脚本参数**：命令行参数输入框，支持从已保存的参数预设中快速加载（详见第三章）
- **超时时间**：最长执行时间，单位秒，超时后自动强制终止进程
- **执行用户**：选择以哪个系统用户身份运行脚本（如 slurm、admin）
  - `root` 只有在目标服务器开启「脚本 root 执行」后才会出现在下拉项中
  - 配置入口：`服务器管理 -> 服务器详情 -> 脚本 root 执行`

### 执行触发
- 配置完成后点击「执行」按钮，向后端发起执行请求
- 执行期间「执行」按钮替换为「终止」按钮，点击可强制结束进程
- 执行完成后按钮恢复为「执行」，日志面板显示退出码

### 实时进度展示
- 执行开始后，页面底部或右侧显示「执行日志」面板
- 使用 **WebSocket** 或 **SSE（Server-Sent Events）** 实时推送日志行
- 日志面板要求：
  - 自动滚动到最新日志行（可手动暂停自动滚动）
  - 区分 stdout（白色）和 stderr（红色/橙色）
  - 显示每行日志的时间戳（`HH:mm:ss`）
  - 执行完成后显示退出码（`Exit code: 0` 绿色 / 非 0 红色）
  - 提供「清空日志」和「复制全部日志」按钮
- 执行期间「执行」按钮变为「终止」按钮，支持强制结束进程

## 三、脚本参数预设

### 功能说明
脚本参数支持「保存为预设」，每个脚本独立维护自己的预设列表，
存储在数据库 `script_param_presets` 表中。

**交互流程：**
1. 用户在参数输入框中输入参数后，点击「保存」图标
2. 弹出命名框，输入预设名称（如「生产环境」「调试模式」）后确认
3. 保存成功后，预设出现在该脚本的预设列表中
4. 下次打开该脚本，前端自动请求此脚本的预设列表并展示
5. 点击某条预设，参数输入框自动填入对应参数值，同时更新该预设的 `last_used_at`
6. 每条预设提供「删除」按钮，删除前弹出确认

**预设列表展示：**
- 按 `last_used_at` 降序排列（最近使用的排最前）
- 每条显示：预设名称、参数内容（过长截断显示）、上次使用时间
- 当前已加载的预设高亮显示

## 四、接口设计（后端 API）

```
# ── 脚本 CRUD ────────────────────────────────────────────────
GET    /api/scripts                       # 获取脚本列表
POST   /api/scripts/upload                # 上传脚本（multipart/form-data）
GET    /api/scripts/:id                   # 获取脚本内容
PUT    /api/scripts/:id                   # 更新脚本内容
DELETE /api/scripts/:id                   # 删除脚本

# ── 脚本执行 ─────────────────────────────────────────────────
POST   /api/scripts/:id/execute           # 执行脚本，返回 task_id
DELETE /api/tasks/:task_id                # 终止执行中的脚本

# ── 实时日志（二选一） ────────────────────────────────────────
WS     /ws/tasks/:task_id/log             # WebSocket 实时推送
GET    /api/tasks/:task_id/log/stream     # SSE 流式推送

# ── 参数预设 ─────────────────────────────────────────────────
GET    /api/scripts/:id/presets           # 获取该脚本的所有参数预设（按 last_used_at 降序）
POST   /api/scripts/:id/presets           # 新建预设
PUT    /api/scripts/:id/presets/:pid      # 修改预设名称或参数内容
DELETE /api/scripts/:id/presets/:pid      # 删除预设
POST   /api/scripts/:id/presets/:pid/use  # 更新 last_used_at（加载预设时调用）

# ── 服务器列表 ───────────────────────────────────────────────
GET    /api/servers                       # 获取已注册服务器列表（含在线状态）
```

## 五、数据存储

脚本文件保存到服务器指定目录（如 `/opt/app/scripts/`），数据库存储元信息和预设数据。

### 数据库表结构

**`scripts` 表** — 脚本元信息
```sql
CREATE TABLE scripts (
    id          INT PRIMARY KEY AUTO_INCREMENT,
    name        VARCHAR(128) NOT NULL,          -- 脚本名称
    filename    VARCHAR(256) NOT NULL,          -- 实际文件名
    description TEXT,                           -- 描述（可为空）
    file_path   VARCHAR(512) NOT NULL,          -- 服务器存储路径
    created_at  DATETIME NOT NULL DEFAULT NOW(),
    updated_at  DATETIME NOT NULL DEFAULT NOW() ON UPDATE NOW()
);
```

**`script_param_presets` 表** — 参数预设（核心表）
```sql
CREATE TABLE script_param_presets (
    id           INT PRIMARY KEY AUTO_INCREMENT,
    script_id    INT NOT NULL,                  -- 关联的脚本 ID（外键）
    name         VARCHAR(64) NOT NULL,          -- 预设名称，如「生产环境编译」
    args         TEXT NOT NULL,                 -- 参数内容，如「--env prod --jobs 15」
    last_used_at DATETIME DEFAULT NULL,         -- 最近一次加载时间，NULL 表示从未使用
    created_at   DATETIME NOT NULL DEFAULT NOW(),
    FOREIGN KEY (script_id) REFERENCES scripts(id) ON DELETE CASCADE
);
-- 索引：按脚本查询预设时按最近使用排序
CREATE INDEX idx_presets_script_used ON script_param_presets (script_id, last_used_at DESC);
```

**`script_tasks` 表** — 执行历史
```sql
CREATE TABLE script_tasks (
    task_id     CHAR(36) PRIMARY KEY,           -- UUID
    script_id   INT NOT NULL,
    server_id   INT NOT NULL,                   -- 在哪台服务器执行
    cwd         VARCHAR(512) NOT NULL,          -- 工作目录
    args        TEXT,                           -- 本次执行的参数
    run_as      VARCHAR(64),                    -- 执行用户
    exit_code   INT DEFAULT NULL,               -- 退出码，执行中为 NULL
    started_at  DATETIME NOT NULL DEFAULT NOW(),
    finished_at DATETIME DEFAULT NULL,
    FOREIGN KEY (script_id) REFERENCES scripts(id) ON DELETE CASCADE
);
```

**`servers` 表** — 已注册服务器
```sql
CREATE TABLE servers (
    id          INT PRIMARY KEY AUTO_INCREMENT,
    name        VARCHAR(64) NOT NULL,           -- 显示名称，如「node01」
    host        VARCHAR(128) NOT NULL,          -- IP 或域名
    port        INT NOT NULL DEFAULT 22,
    username    VARCHAR(64) NOT NULL DEFAULT 'slurm',
    description VARCHAR(256),
    created_at  DATETIME NOT NULL DEFAULT NOW()
);
```

## 六、安全要求

- 上传文件类型白名单校验（仅允许 `.sh` `.bash` `.py`）
- 脚本上传不再使用固定 1MB 限制
- 执行用户可配置；默认不展示 `root`
- 如需允许 `root` 执行，必须在对应服务器详情页开启「脚本 root 执行」
- 可执行目录白名单：仅允许在该服务器「允许的工作目录」中执行
- 脚本执行优先使用用途为 `script` 的目录，`general` 目录作为通用目录可复用
- 超时机制：所有脚本执行均强制设置最长超时，防止进程挂死

## 八、文件产物对接

### 对接原则

与 Apptainer、Benchmark 模块保持完全一致的模式：
- 脚本模块**不维护任何桥接表**，不持有 artifact 外键
- 真实文件先落到 `DATA_ROOT`，或只记录远程路径
- 执行成功后调用 `apps.artifacts.utils.create_artifact()` 写入元数据
- 产物通过 `job_type + job_id + artifact_type` 软关联，不用强外键
- 前端直接复用已有的 `GET /api/v1/artifacts/` 接口过滤查询，**无需新增产物相关接口**

### Step 2：后端任务完成后调用 `create_artifact()`

参考 `backend/apps/apptainer/tasks.py:114` 和 `backend/apps/benchmark/tasks.py:110`，
在 `backend/apps/scripts/tasks.py` 的执行任务完成回调中加入：

```python
from apps.artifacts.utils import create_artifact
import os, shutil

DATA_ROOT = settings.DATA_ROOT  # 已有配置

def collect_script_artifacts(task, new_files: list[dict]):
    """
    task:       ScriptTask 实例
    new_files:  执行前后 diff 得到的新增/变更文件列表
                每项格式: {"name": str, "remote_path": str, "is_remote": bool}
    """

    # ── 1. 执行日志（本地文件，保存到 DATA_ROOT）────────────────
    log_dir = os.path.join(DATA_ROOT, "artifacts", "scripts", "logs")
    os.makedirs(log_dir, exist_ok=True)
    log_path = os.path.join(log_dir, f"task_{task.task_id}.log")
    # 将内存中的日志内容写入文件（具体写法视日志存储方式而定）
    with open(log_path, "w") as f:
        f.write(task.full_log)

    create_artifact(
        project_id    = task.project_id,
        job_type      = "script_execution",
        job_id        = str(task.task_id),
        artifact_type = "script_log",
        file_name     = f"task_{task.task_id}.log",
        local_path    = log_path,
        local_file    = True,           # 自动计算 file_size 和 SHA256
    )

    # ── 2. 脚本执行产物（diff 发现的新增文件）────────────────────
    out_dir = os.path.join(DATA_ROOT, "artifacts", "scripts", "outputs", f"task_{task.task_id}")
    os.makedirs(out_dir, exist_ok=True)

    for f in new_files:
        if f["is_remote"]:
            # 远程文件：只记录路径，不下载（同 sif_path_record 的做法）
            create_artifact(
                project_id    = task.project_id,
                job_type      = "script_execution",
                job_id        = str(task.task_id),
                artifact_type = "script_output",
                file_name     = f["name"],
                storage_path  = f["remote_path"],   # 远程绝对路径
                local_file    = False,
            )
        else:
            # 本地文件：复制到 DATA_ROOT 后登记
            dst = os.path.join(out_dir, f["name"])
            shutil.copy2(f["remote_path"], dst)
            create_artifact(
                project_id    = task.project_id,
                job_type      = "script_execution",
                job_id        = str(task.task_id),
                artifact_type = "script_output",
                file_name     = f["name"],
                local_path    = dst,
                local_file    = True,
            )
```

### Step 3：执行前后目录快照 Diff（产物发现逻辑）

无需改动脚本本身，由后端在执行前后自动扫描：

```python
def snapshot_dir(path: str) -> dict:
    """返回目录下所有文件的 {相对路径: (mtime, size)} 字典"""
    result = {}
    for root, _, files in os.walk(path):
        for name in files:
            full = os.path.join(root, name)
            rel  = os.path.relpath(full, path)
            if _should_ignore(rel):     # 过滤 .tmp .lock __pycache__ 等
                continue
            stat = os.stat(full)
            result[rel] = (stat.st_mtime, stat.st_size)
    return result

def diff_snapshots(before: dict, after: dict) -> list[str]:
    """返回新增或内容变更的文件相对路径列表"""
    new_files = []
    for path, (mtime, size) in after.items():
        if path not in before or before[path] != (mtime, size):
            new_files.append(path)
    return new_files
```

扫描范围：`cwd` + 用户在执行配置中额外指定的「输出目录」（见执行配置字段）。
忽略规则：`.tmp` `.lock` `.swp` `__pycache__/` `.git/` `*.pyc`。

| 优先级 | 功能 |
